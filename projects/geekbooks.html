Feature-to-product translation

The SAFe ART backlog, now reflected in the storefront.

The first slice implements a believable static version of the highest-value flows, while the risky backend-heavy features stay clearly marked.

Priority 1 • implemented as prototype

Flexible search

Search by title, author, genre, publisher, description, and badge using the test database.

Priority 2 • implemented as prototype

Shopping cart

Add/remove books, quantity totals, subtotal, clear cart, save for later, localStorage persistence.

Priority 3 • mocked only

Credit card purchase

Checkout is a demo note only. Real payments need a PCI-safe payment provider and backend.

Priority 4 • mocked only

Shipping methods

Shipping quote remains future work needing carrier APIs and address validation.

Priority 5+ • future backend

Profiles, ratings, comments

Visible in book data and roadmap, but verified purchasers, login, email reminders, and GDPR workflows need services.

Honest implementation notes

What can and cannot be done here.

GitHub Pages can carry a surprisingly convincing storefront prototype. It still cannot safely hold your credit card, which is rude but correct.

✅ Working static features

  • Static JSON test database
  • Client-side catalogue search/filter/sort
  • Book detail modal
  • Cart quantity/subtotal in localStorage
  • Save for later list

⚠️ Needs backend/services

  • User accounts and secure login
  • PCI-compliant credit-card purchase
  • Shipping quotes from carrier APIs
  • Verified purchaser ratings/comments
  • GDPR deletion and email notifications
Diego design verdict

GeekBooks is now a real static bookstore prototype.
