Prioritised feature map

The ART backlog, translated into product experience.

Features are ordered from the simulation handout. The first four form the commerce spine; later features deepen trust, discovery, and community.

Priority 1

Flexible search

One field for author, title, genre, suggestions, misspellings, and match-ranked results.

  • Match API
  • Recommendations
  • Internationalisation spike
Priority 2

Shopping cart

Persistent cart access, quantity edits, remove, save for later, subtotal, and shipping quote.

  • Cart count
  • AJAX update spike
  • Saved items
Priority 3

Credit card purchase

Payment details, saved cards, shipping info, previous purchases, and PCI-safe handling.

  • Encryption spike
  • Memory cleanup spike
  • Reorder journey
Priority 4

Shipping methods

Compare carrier options by price, speed, and estimated date.

  • UPS/FedEx API spike
  • Shipping preferences
  • Carrier estimates
Priority 5

Profile management

Login, personal info, addresses, cards, nicknames, avatars, and right-to-be-forgotten flows.

  • A/B profile creation
  • Avatar component spike
  • Secure login
Priority 6

Book detail

Rich book pages with cover, author bio, description, genre, publishing info, ratings, and comments.

Priority 7

Book list sorting

Sort lists by title, author, price, rating, release date, plus pagination and page-size options.

Priority 8

Book browsing

Browse genres, Tech Valley Times best sellers, top sellers, rating leaders, and time-based trends.

Priority 9

Book rating

Five-star ratings from verified purchasers, with nickname/privacy controls and rating reminders.

Priority 10

Commenting

Purchaser-only comments, rich text, character indicators, editing windows, spam protection, and GDPR deletion.

Customer lens

Personas hiding inside the stories.

🔎

Knows roughly what they want and needs forgiving, fast search.

🛍️

Book shopper

Compares options, curates a cart, and saves candidates for later.

💳

Book purchaser

Needs secure checkout, profiles, shipping choices, and trust.

📚

Book browser

Explores genres, best sellers, ratings, and delightful rabbit holes.

⭐

Book rater

Shares trusted post-purchase signal with privacy control.

💬

Book commentator

Writes reviews, edits/removes them, and expects safety from spam chaos.

PI planning view

MVP slice vs future increments.

For a first release, the site should prove the browse-search-cart loop before trying to swallow PCI, GDPR, carrier integrations, and the entire internet.

🚀 PI 1 MVP

  1. Flexible search shell
  2. Book detail cards
  3. Browsing by genre/bestseller/rating
  4. Cart preview with save for later
  5. Static checkout intent page

🧪 Enablers first

  • Match algorithm/API design
  • AJAX cart updates
  • Shipping API feasibility
  • PCI/payment provider architecture
  • Profile UX A/B prototype

🌱 Later PIs

  • Real accounts and login
  • Saved cards via payment tokenisation
  • Verified purchaser ratings/comments
  • Email reminders
  • GDPR erase/export workflows

Honest implementation notes

What this static site can and cannot do.

GitHub Pages is brilliant for prototypes and storytelling. It is not a PCI-compliant ecommerce platform. Tiny but important distinction.

✅ Feasible here

  • Static product/marketing page
  • Client-side search mockup over sample books
  • Visual cart prototype
  • Roadmap and ART communication
  • Feature priority board and persona framing

⚠️ Needs real services

  • User accounts, secure login, password handling
  • PCI-compliant credit-card purchase and saved cards
  • Shipping quotes from carrier APIs
  • Verified purchaser-only ratings/comments
  • GDPR deletion, email reminders, spam protection
Diego design verdict

GeekBooks is ready for a glossy PI-planning demo.

The first compelling slice is search + browse + book detail + cart intent. Payments, profiles, comments, and ratings should be treated as backend-backed future increments with serious security architecture, not magic confetti.
