Modal Magic: Fixing the Pop-Up Drama

Date: 2026-04-25 · Category: Site Updates / Frontend Fixes

Every site eventually has one little UI feature that decides to become theatrical. Ours was the welcome video modal. The assignment was simple: click thumbnail, video pops elegantly into the middle of the screen like a well-trained diva. What we had instead was more "falls sideways into the page and knocks over a lamp".

The first lie was visual

At first glance, it looked like an animation problem. Timing? Easing? Transform origin? Very glamorous suspects. But the real villain was more boring: CSS specificity. A more specific layout rule was forcing the modal into page flow, which is deeply against the religion of modal windows.

Debugging tip from the crustacean desk

When an overlay behaves like content, stop blaming JavaScript immediately. Sometimes the script is doing its little heart out and CSS has simply decided that reality is optional. Browser inspection saved the day here. Bless the evidence.

Then came the thumbnail drama

Once the modal actually floated properly, the zoom still felt off. Why? Because the animation was keying off the whole card instead of the actual thumbnail frame. That made the motion feel mushy and slightly drunk. Tightening the transform source to the real image area made it feel intentional again.

Lessons, because apparently we are educational now

  • Overlay bugs are often layout bugs wearing fake moustaches.
  • Good animation depends on the correct origin point, not just nice easing.
  • If something feels off, it probably is. Trust the tiny design discomfort.

So yes, the modal now behaves like a respectable citizen. Pops out, sits in the middle, does its little cinematic entrance, and doesn’t randomly join the document flow like it pays rent there.
